# zmeel/templates

The canonical catalog of role templates for [zmeel](https://zmeel.ai) companies. Maintained by the zmeel team.

Each subdirectory under `templates/` is one role template. When a user hires from a template in the app, the zmeel server reads the files here and populates the new agent's brain + skills.

## Structure

```
templates/
  <id>/
    manifest.yaml        # metadata (label, category, icon, defaultTools, systemPrompt, ...)
    soul.md              # agent persona
    procedures.md        # operating procedures
    heartbeat.md         # per-heartbeat checklist
    memory.md            # memory layer prompt
    tools.md             # tool notes
    learning.md          # learning layer prompt
    skills/
      <slug>/
        SKILL.md         # optional, zero or more per template
registry.json            # auto-generated by CI; do not edit by hand
```

## Editing

1. Open the template folder in the GitHub web editor.
2. Commit to `main`.
3. A push webhook notifies the zmeel server; the edit appears in new hires within seconds.
4. The `Build registry` workflow regenerates `registry.json`.

For existing agents hired from this template, their brain files are a point-in-time copy. Re-hire to pick up newer content.

## Adding a template

1. Create `templates/<id>/` with the files listed above. `id` must match the folder name.
2. `manifest.yaml` must be valid per the schema enforced by the zmeel server (`server/src/services/template-manifest.ts`).
3. Push to main. The workflow regenerates `registry.json`. No PR review gate -- this is an internal, zmeel-maintained catalog.

## Substitution

At hire time, the server substitutes `{company}` and `{name}` in the brain files and `systemPrompt`. Keep templates generic -- they're reusable across companies.

## Skills

Each template may ship zero or more skill bundles under `skills/<slug>/SKILL.md`. These are imported into the hiring company's `company_skills` table on first hire and attached to the new agent's `adapterConfig.zmeelSkillSync.desiredSkills`.
