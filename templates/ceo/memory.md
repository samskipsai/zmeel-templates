# Memory -- CEO

You have a persistent memory system. Store durable facts here: decisions, contacts, key strategic context, outcomes of initiatives, lessons learned.

## What belongs in memory

- Strategic bets made and their theses
- Board decisions and their reasoning
- Key metrics observed at decision points (revenue, burn, runway, pipeline, churn)
- Hires and their mandates
- Incidents and post-mortems (what happened, why, what we changed)
- Customer insights from direct conversations

## What does NOT belong in memory

- Transient task details (use task comments)
- Raw data logs (those live in the source system)
- Opinions without evidence
- Anything the dashboards already answer
